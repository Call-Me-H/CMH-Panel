Welcome to CMH Panel Manual خوش آمدید به راهنمای پنل CMH

This guide explains exactly how to use every feature inside the CMH Panel Dashboard. The panel is divided into three main tabs: Profiles (for getting connection links), Users (for managing clients and traffic), and Settings (for system configuration). این راهنما دقیقاً نحوه استفاده از تمامی امکانات داخل داشبورد CMH را توضیح می‌دهد. پنل به سه بخش اصلی تقسیم شده است: تب Profiles (برای دریافت لینک‌های اتصال کلاینت)، تب Users (برای مدیریت مشترکین و ترافیک آن‌ها)، و تب Settings (برای پیکربندی هسته سیستم).

🌍 Profiles Tab تب پروفایل‌ها (Profiles)

Once you create a user in the "Users" tab, they will appear here. This tab is used to distribute configurations to your clients. Each user gets a dedicated card with the following options: هنگامی که یک کاربر جدید ایجاد می‌کنید، کارت مربوط به او در این تب ظاهر می‌شود. این بخش برای دریافت کانفیگ و ارائه آن به مشتریان استفاده می‌شود. هر کاربر گزینه‌های زیر را برای دریافت کانفیگ دارد:

🌐 Open Sub Page 🌐 صفحه ساب اختصاصی (Sub Page)

Opens a beautiful, client-facing webpage. The client can view their Total & Daily usage, remaining days, and easily copy or scan their configs from there. This is the link you should send to your customers. این دکمه یک صفحه وب بسیار زیبا و اختصاصی برای کاربر باز می‌کند. کلاینت می‌تواند میزان مصرف کل، مصرف روزانه و روزهای باقی‌مانده خود را ببیند و کانفیگ‌ها را مستقیماً از آنجا کپی یا اسکن کند. (این لینکی است که باید به مشتری بدهید)

Clash Config کانفیگ کلش (Clash)

Generates a YAML file formatted specifically for Clash clients. It automatically includes Fallback and Auto Best Ping (URL-Test) proxy groups for all your IPs and Ports. یک فایل استاندارد YAML مخصوص کلاینت‌های مبتنی بر کلش تولید می‌کند. این فایل به‌صورت خودکار دارای گروه‌های هوشمند Fallback (جایگزین خودکار آی‌پی قطع شده) و Auto Best Ping است.

Normal Config کانفیگ معمولی (Normal)

Generates a standard Base64 encoded subscription link (vless:// and trojan:// links). Best for v2rayNG, v2rayN, and Shadowrocket. لینک‌های استاندارد VLESS و Trojan را در قالب یک فرمت Base64 ارائه می‌دهد. این لینک بهترین گزینه برای برنامه‌هایی مانند v2rayNG، v2rayN و Shadowrocket است.

JSON Config کانفیگ جیسون (JSON)

Generates a raw JSON format array of Outbounds. Useful for advanced clients like Sing-Box or custom routing setups. یک خروجی کاملاً خام JSON شامل تمامی Outboundها ارائه می‌دهد. مناسب برای ایمپورت در کلاینت‌های بسیار حرفه‌ای مانند Sing-Box یا روتینگ‌های شخصی.

👥 Users Tab تب کاربران (Users)

This tab is your control center for managing subscribers. You can add users, set precise traffic/time limits, and monitor their real-time usage. این تب مرکز مدیریت مشترکین شماست. می‌توانید کاربران جدید بسازید، محدودیت‌های دقیق حجمی و زمانی اعمال کنید و مصرف لحظه‌ای آن‌ها را زیر نظر بگیرید.

💡 How Traffic is Calculated 💡 ترافیک چگونه محاسبه می‌شود؟

Since Cloudflare Workers do not provide byte-level tracking per socket, the panel counts Requests. The system uses a rough estimation where 1 GB ≈ 6,000 Requests. When you set a limit in GB, the panel automatically converts and enforces it in Requests. از آنجا که محیط Cloudflare Workers قابلیت شمارش دقیق بایت‌های عبوری در سطح سوکت را ندارد، سیستم تعداد ریکوئست‌ها (Requests) را می‌شمارد. پنل به صورت تقریبی در نظر می‌گیرد که هر ۱ گیگابایت معادل حدود ۶,۰۰۰ ریکوئست است. وقتی شما محدودیت را بر اساس گیگابایت (GB) تنظیم می‌کنید، سیستم خودکار آن را به ریکوئست تبدیل و اعمال می‌کند.

Creating / Editing a User ساخت / ویرایش کاربر

Field نام فیلد Description توضیحات
Username The name of the user (e.g., John_Doe). This will be used in the config names. نام کاربری. این نام در اسم کانفیگ‌ها و ساب‌اسکریپشن نمایش داده می‌شود.
Total Limit Maximum traffic allowed for the entire lifespan of the user. Choose GB or Reqs. Leave empty for Unlimited. محدودیت حجم کل (برای کل دوره کاربر). می‌توانید بر اساس GB یا Req تنظیم کنید. اگر خالی بماند، نامحدود خواهد بود.
Daily Limit Maximum traffic allowed per day. Resets automatically at UTC 00:00. Leave empty for Unlimited. محدودیت حجم روزانه. این حجم هر روز (بر اساس ساعت UTC) صفر می‌شود. اگر خالی بماند، نامحدود است.
Expiration (Days) The number of days the user has access. Countdown starts immediately. Leave empty for no expiry. تعداد روزهای اعتبار کاربر. به محض ثبت، شمارش معکوس آغاز می‌شود. اگر خالی بگذارید، بدون انقضا خواهد بود.

User Actions عملیات مدیریت کاربران (Actions)

  • 🔄 Reset: Instantly clears the user's Total and Daily usage back to zero. 🔄 Reset (ریست): مصرف کل و مصرف روزانه کاربر را فوراً صفر می‌کند.
  • ⏸️ Disable / ▶️ Enable: Manually blocks or allows the user to connect. (Users are auto-disabled if they hit limits). ⏸️ Disable / ▶️ Enable (تغییر وضعیت): مسدودسازی دستی یا فعال‌سازی مجدد اتصال کاربر. (اگر کاربری به سقف مصرف برسد، سیستم به صورت خودکار او را Disable می‌کند).
  • ✏️ Edit: Change limits or add more days. ✏️ Edit (ویرایش): تغییر نام، حجم مجاز یا افزودن روزهای بیشتر.

⚙️ Settings Tab تب تنظیمات (Settings)

The Settings tab configures how the entire system behaves. It is divided into logical sections. After making changes, always click "Save Configuration" at the bottom. تب تنظیمات برای پیکربندی رفتار کل سیستم استفاده می‌شود و به بخش‌های مجزا تقسیم شده است. توجه: پس از اعمال هر تغییری، حتماً دکمه "Save Configuration" (ذخیره تنظیمات) در پایین صفحه را بزنید.

1. Cluster Setup (Master/Edge) ۱. کلاستر و توزیع بار (Master/Edge)

Cloudflare limits free accounts to 100,000 requests per day per worker. To bypass this, you can deploy multiple proxy workers (Edges) that report back to a central Database worker (Master). اکانت‌های رایگان کلودفلر روزانه محدود به ۱۰۰,۰۰۰ ریکوئست به ازای هر ورکر هستند. برای دور زدن این محدودیت، سیستم اجازه می‌دهد چندین ورکر فرعی (Edge) به یک دیتابیس مرکزی (Master) متصل شوند.

Master Node 👑 نود اصلی (Master Node) 👑 Your primary worker with the D1 Database bound. It hosts the Dashboard and Subscription pages. If you select this, you can enter URLs of other workers in the "Edge Nodes" field below to balance load. این ورکر اصلی شماست که دیتابیس D1 به آن متصل است و داشبورد روی آن بالا می‌آید. اگر این حالت را انتخاب کنید، می‌توانید آدرس ورکرهای فرعی خود را در فیلد "Edge Nodes" وارد کنید تا ترافیک روی آن‌ها تقسیم شود.
Edge Node 📡 نود لبه (Edge Node) 📡 A secondary worker (no D1 required). If selected, the UI hides users/profiles. You must enter the Master API URL and Master Admin Password so this Edge can fetch users and report its traffic back to the Master. ورکر ثانویه است که نیازی به دیتابیس ندارد و داشبوردش قفل می‌شود. در این حالت باید آدرس ورکر Master و رمز عبور ادمین Master را وارد کنید. این ورکر لیست کاربران را از نود اصلی می‌گیرد و ترافیک مصرفی را برای محاسبه به نود اصلی می‌فرستد.

2. General Configurations ۲. تنظیمات عمومی (General) Global Pause توقف کل سیستم

The "Global System Status" button at the top acts as a kill-switch to immediately drop all connections to this node. دکمه "Global System Status" در بالای این بخش به عنوان یک Kill-Switch (کلید قطع اضطراری) عمل می‌کند و در صورت فعال‌سازی، تمام اتصالات ورکر فوراً قطع می‌شوند.

Protocol پروتکل (Protocol) Choose VLESS, Trojan, or Both. Subscriptions will generate accordingly. انتخاب بین VLESS، Trojan یا هردو. ساب‌اسکریپشن‌ها بر اساس این انتخاب ساخته می‌شوند.
Ports پورت‌ها (Ports) Select one or multiple ports (Hold Ctrl/Cmd).
HTTPS (TLS): Enables WebSockets over TLS.
HTTP (Non-TLS): Enables raw WebSockets (useful behind CDNs). می‌توانید چند پورت را با نگه‌داشتن کلید Ctrl انتخاب کنید.
گروه TLS (مثل 443): تنظیمات امنیتی TLS را اعمال می‌کند.
گروه Non-TLS (مثل 80): مناسب برای استفاده در پشت CDNها بدون TLS.
Clean IPs Enter proxy IPs, CDN IPs, or domains (one per line or comma-separated). The panel will generate a separate config for each IP combined with each Port. آی‌پی‌های تمیز، آی‌پی سرورهای CDN یا دامنه‌های خود را وارد کنید (هر خط یک آی‌پی یا با کاما). پنل به ازای هر آی‌پی ضربدر هر پورت، یک کانفیگ تولید می‌کند.
Name Strategy الگوی نام‌گذاری (Name Strategy) Customize how configs are named in the client app. Available variables:
{PROTOCOL} : VLESS or Trojan
{USER} : Username
{PORT} : Port Number
{IP} : The Clean IP
{PREFIX} : Text from Name Prefix field. نحوه نمایش نام کانفیگ‌ها در اپلیکیشن کاربر را تغییر می‌دهد. متغیرهای مجاز:
{PROTOCOL} : نام پروتکل
{USER} : نام کاربری
{PORT} : شماره پورت
{IP} : آی‌پی تمیز
{PREFIX} : متن دلخواه وارد شده در فیلد Name Prefix
Resolver IP / DoH Used if the worker needs to resolve DNS queries internally (e.g., 1.1.1.1). اگر کاربر بخواهد به جای آدرس عددی، یک دامنه را پراکسی کند، ورکر برای پیدا کردن IP هدف از این سرویس (مثل 1.1.1.1) استفاده می‌کند.
Proxy ip آپروکسی آیپی (Proxy ip) By entering a custom server address ALL traffic will be forced through it. If left blank, it uses Cloudflare's dynamic IPs. در صورت وارد کردن یک آدرس سرور سفارشی، تمام ترافیک به‌اجبار از طریق آن هدایت می‌شود. در غیر این صورت ، از آی‌پی‌های پویای کلودفلر استفاده خواهد شد.
Camouflage Host هاست استتار (Camouflage Host) If someone visits your worker URL directly without the correct path, they will be redirected/proxied to this harmless website (e.g., hcaptcha.com). اگر ربات‌های فیلترینگ یا افراد ناشناس، لینک ورکر شما را بدون مسیر (Path) صحیح باز کنند، به این سایت بی‌خطر (مثلاً hcaptcha.com) ریدایرکت می‌شوند تا ماهیت ورکر مخفی بماند.

3. Smart Fragment (DPI Bypass) ۳. فرگمنت هوشمند (بایپس فیلترینگ / DPI)

Fragmentation splits the initial TLS handshake into smaller pieces. This prevents Deep Packet Inspection (DPI) firewalls from reading the Server Name Indication (SNI) and blocking the connection. If enabled, this is automatically applied to Clash and JSON generated configs for TLS ports. مکانیزم فرگمنت، بسته اولیه TLS (موسوم به Client Hello) را به قطعات کوچکتر تقسیم می‌کند تا فایروال‌های سیستم فیلترینگ (DPI) نتوانند نام دامنه (SNI) را بخوانند و اتصال را مسدود کنند. این تنظیمات مستقیماً درون کانفیگ‌های Clash و JSON خروجی قرار می‌گیرند.

Enable Fragment Toggle switch. Turns the feature on or off globally for generated subscriptions. کلید روشن/خاموش. اعمال یا عدم اعمال این ویژگی در ساب‌اسکریپشن‌ها را کنترل می‌کند.
Size (Length) طول (Size) e.g., 10-100. The random number of bytes to send in each packet slice. مثال: 10-100. تعداد بایت‌های رندوم در هر تکه از بسته ارسال شده.
Interval فاصله زمانی (Interval) e.g., 1-10. The delay in milliseconds between sending each fragment. مثال: 1-10. وقفه زمانی (به میلی‌ثانیه) بین ارسال هر قطعه.
Step (Packets) بسته هدف (Step) Usually set to tlshello. This tells the client to only fragment the initial handshake, keeping the rest of the stream fast and unfragmented. معمولاً روی tlshello تنظیم می‌شود. به کلاینت می‌گوید که فقط مرحله دست‌دهی اولیه (Handshake) را تکه‌تکه کند تا سرعت کلی اتصال افت نکند.

4. Telegram Bot Integration ۴. اتصال ربات تلگرام

You can manage users, check usage, and pause the system entirely via a Telegram Bot without needing to log into this Dashboard. با اتصال ربات، می‌توانید بدون نیاز به ورود به داشبورد وب، کارهای پنل مانند ساخت کاربر، دریافت وضعیت سیستم، مسدودسازی کاربران و مشاهده حجم مصرفی را مستقیماً از تلگرام انجام دهید.

Step 1: Create a bot using @BotFather in Telegram and copy the HTTP API Token. Paste it in "Bot Token". مرحله ۱: در تلگرام به ربات @BotFather رفته و یک ربات جدید بسازید. توکن HTTP API داده شده را کپی کرده و در فیلد "Bot Token" وارد کنید.

Step 2: Find your personal Telegram Account ID using a bot like @userinfobot. Paste it in "Admin ID" (so only YOU can control it). مرحله ۲: آیدی عددی اکانت تلگرام شخصی خودتان را (از طریق ربات‌هایی مثل @userinfobot) پیدا کنید و در فیلد "Admin ID" قرار دهید (تا هیچ‌کس جز شما نتواند به ربات دستور بدهد).

Step 3: Click Save Configuration. Then go to your bot and send /start. مرحله ۳: در پایین صفحه داشبورد روی "ذخیره تنظیمات" کلیک کنید. سپس به ربات خود در تلگرام رفته و /start را ارسال کنید.

5. Cloudflare Analytics (Multi-Account) ۵. آمار کلاودفلر (پشتیبانی از چند اکانت)

Since Cloudflare Workers are limited to 100,000 requests per day, it is crucial to monitor this. You can link multiple Cloudflare accounts (e.g., your Master account and 3 Edge accounts) here. با توجه به محدودیت روزانه ۱۰۰ هزار ریکوئست، بررسی میزان درخواست‌های مصرف شده بسیار مهم است. در این بخش می‌توانید چندین اکانت کلاودفلر (مثلاً اکانت Master و سه اکانت Edge) را همزمان به سیستم متصل کنید.

Once you add accounts here, you can click the "CF Usage" button in your Telegram Bot. The bot will instantly fetch and display a beautiful progress bar showing the daily 100k limit usage for all linked accounts at once! پس از افزودن اکانت‌ها در اینجا، با زدن دکمه "CF Usage" در ربات تلگرام، ربات در همان لحظه میزان ریکوئست مصرف شده نسبت به سقف ۱۰۰ هزارتایی را برای تک‌تک اکانت‌های متصل شده به صورت نمودار میله‌ای به شما نشان می‌دهد!

Node/Account Name نام اکانت/نود A nickname you give it (e.g., "Main Account" or "Edge 1"). یک نام دلخواه برای تشخیص راحت‌تر (مثلا "اکانت اصلی" یا "نود دوم").
Account ID Found in your Cloudflare Dashboard URL: dash.cloudflare.com/[THIS_LONG_STRING]/workers این شناسه را می‌توانید در آدرس URL داشبورد کلاودفلر خود پیدا کنید: dash.cloudflare.com/[THIS_LONG_STRING]/workers
API Token Go to CF Profile -> API Tokens -> Create Token. Use "Read all resources" template, or create custom with Account/Worker Metrics Read permission. در کلاودفلر به بخش My Profile -> API Tokens بروید. می‌توانید از قالب "Read all resources" استفاده کنید یا یک توکن اختصاصی بسازید که دسترسی خواندن (Read) برای Account/Worker Metrics داشته باشد.
